# Task: Environment Configuration

## Description

Tạo file cấu hình môi trường (.env) cho frontend để quản lý API base URL. Điều này cho phép dễ dàng chuyển đổi giữa các môi trường development, staging, và production mà không cần hardcode URLs trong code.

## Acceptance Criteria

- [x] File `.env` được tạo trong thư mục `frontend/` với biến `VITE_API_BASE_URL=http://127.0.0.1:8001`
- [x] File `.env.example` được cập nhật với template và documentation
- [x] Documentation rõ ràng về cách cấu hình cho các môi trường khác nhau
- [x] Verify rằng Vite có thể đọc được environment variable qua `import.meta.env.VITE_API_BASE_URL`

## Technical Details

**Files to modify:**
- `frontend/.env.example` - Thêm VITE_API_BASE_URL với example value

**Files to create:**
- `frontend/.env` - Local configuration file (không commit vào git)

**Implementation approach:**
1. Tạo file `frontend/.env` với content:
   ```
   VITE_API_BASE_URL=http://127.0.0.1:8001
   ```

2. Cập nhật `frontend/.env.example` với:
   ```
   # API Base URL
   # Development: http://127.0.0.1:8001
   # Staging: https://staging-api.fhs-prosight.com
   # Production: https://api.fhs-prosight.com
   VITE_API_BASE_URL=http://127.0.0.1:8001
   ```

3. Verify `.gitignore` đã có `.env` (Vuexy template thường đã có sẵn)

**Key considerations:**
- Vite tự động load `.env` files khi start dev server
- Chỉ variables với prefix `VITE_` mới được expose ra client
- `.env` file không được commit vào git (sensitive data)
- `.env.example` là template cho team members
