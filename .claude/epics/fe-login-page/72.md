# Task: Manual Testing & QA

## Description

Thực hiện manual testing toàn diện cho OAuth login flow, verify tất cả requirements đã được implement đúng, test trên nhiều browsers và devices, và đảm bảo user experience mượt mà.

## Acceptance Criteria

- [x] Google OAuth flow hoàn chỉnh end-to-end
- [x] GitHub OAuth flow hoàn chỉnh end-to-end
- [x] Token persistence sau browser refresh
- [x] Error handling cho tất cả error scenarios
- [x] Dark mode và light mode hiển thị đúng
- [x] Responsive design hoạt động trên mobile/tablet/desktop
- [x] Cross-browser compatibility (Chrome, Firefox, Safari, Edge)
- [x] No console errors
- [x] Performance acceptable (< 2s page load)

## Technical Details

### Test Cases

#### 1. Google OAuth Flow
**Steps:**
1. Navigate to `/login`
2. Click "Đăng nhập với Google" button
3. Verify redirect to Google OAuth consent screen
4. Login với Google account
5. Verify redirect back to `/auth/callback?token=...`
6. Verify loading indicator displays
7. Verify redirect to dashboard/home
8. Open DevTools → Application → Local Storage
9. Verify `auth_token` exists và có correct value
10. Refresh browser (F5)
11. Verify token still in localStorage

**Expected:**
- ✅ Smooth redirect flow
- ✅ Token saved correctly
- ✅ No errors in console
- ✅ User lands on dashboard

#### 2. GitHub OAuth Flow
**Steps:**
1. Clear localStorage (DevTools → Application → Clear Storage)
2. Navigate to `/login`
3. Click "Đăng nhập với GitHub" button
4. Verify redirect to GitHub authorization page
5. Authorize application
6. Verify redirect back to `/auth/callback?token=...`
7. Verify loading indicator displays
8. Verify redirect to dashboard/home
9. Verify `auth_token` in localStorage
10. Refresh browser
11. Verify token persists

**Expected:**
- ✅ Same smooth flow as Google
- ✅ Token saved correctly
- ✅ No console errors

#### 3. Error Scenarios

**Test 3a: User cancels OAuth**
1. Navigate to `/login`
2. Click OAuth button
3. Click "Cancel" or "Deny" on OAuth consent screen
4. Verify redirect back to `/login?error=access_denied`
5. Verify error message displays: "Bạn đã hủy đăng nhập..."
6. Verify error can be dismissed

**Test 3b: Invalid token**
1. Navigate directly to `/auth/callback?token=invalid`
2. Verify redirect to `/login?error=invalid_token`
3. Verify error message displays

**Test 3c: No token**
1. Navigate directly to `/auth/callback`
2. Verify redirect to `/login?error=no_token`
3. Verify error message displays

#### 4. UI/UX Testing

**Test 4a: Dark Mode**
1. Navigate to `/login`
2. Switch to dark mode (Vuexy theme switcher)
3. Verify:
   - Background colors correct
   - Button colors correct (GitHub button especially)
   - Text readable
   - Illustration visible
   - No color contrast issues

**Test 4b: Light Mode**
1. Switch to light mode
2. Verify same criteria as dark mode

**Test 4c: Responsive Design**

*Desktop (> 1280px):*
- ✅ Two-column layout
- ✅ Illustration visible on left
- ✅ Login card on right
- ✅ Buttons full width within card

*Tablet (768px - 1280px):*
- ✅ Two-column layout (might stack on smaller tablets)
- ✅ Readable text
- ✅ Touch-friendly buttons (min 44x44px)

*Mobile (< 768px):*
- ✅ Single column layout
- ✅ Illustration hidden (Vuexy default behavior)
- ✅ Login card full width
- ✅ Buttons full width
- ✅ Text readable
- ✅ No horizontal scroll

#### 5. Browser Compatibility

Test trên các browsers:
- [x] Chrome (latest)
- [x] Firefox (latest)
- [x] Safari (latest) - nếu có Mac
- [x] Edge (latest)

**Verify:**
- OAuth redirect works
- localStorage access works
- No console errors specific to browser
- UI renders correctly

#### 6. Performance Testing

**Metrics to check:**
1. Login page load time: < 2 seconds
   - Open DevTools → Network → Reload page
   - Check "Load" time in bottom status bar
2. OAuth redirect latency: < 100ms from button click
   - Subjective - should feel instant
3. Callback processing: < 500ms
   - Time from landing on `/auth/callback` to redirect to dashboard
