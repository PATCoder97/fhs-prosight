# Task: Add OAuth Callback Route to Router

## Description

Thêm route `/auth/callback` vào Vue Router configuration để map đến auth-callback.vue page. Route này cần có meta flags: layout blank và public (không yêu cầu authentication).

## Acceptance Criteria

- [x] Route `/auth/callback` được thêm vào router configuration
- [x] Route maps to `auth-callback.vue` page
- [x] Meta flags:
  - `layout: 'blank'` - Không hiển thị layout wrapper
  - `public: true` - Không yêu cầu authentication
- [x] Route hoạt động khi navigate đến `/auth/callback`
- [x] Backend có thể redirect đến URL này

## Technical Details

**File to modify:**
- Vuexy sử dụng file-based routing với `unplugin-vue-router`
- Nếu dùng file-based routing: Route tự động tạo từ file structure
- Nếu dùng manual routing: Cần thêm vào `router/index.js` hoặc tương tự

**Check routing approach:**
Vuexy template thường dùng `unplugin-vue-router` - tự động generate routes từ file structure trong `src/pages/`.

**If using file-based routing (unplugin-vue-router):**
- File `src/pages/auth-callback.vue` tự động tạo route `/auth-callback`
- Meta được define trong file với `definePage()`
- Đã implement trong Task 69 ✅

**If using manual routing:**
Add to router configuration:
```javascript
// router/index.js or similar
{
  path: '/auth/callback',
  name: 'auth-callback',
  component: () => import('@/pages/auth-callback.vue'),
  meta: {
    layout: 'blank',
    public: true,
  },
}
```

**Verification steps:**
1. Check if Vuexy uses file-based routing (check `vite.config.js` for `unplugin-vue-router`)
2. If file-based: Verify file created in Task 69 với `definePage()` meta
3. If manual: Add route configuration
4. Test navigation: `router.push('/auth/callback')`
5. Verify meta: `route.meta.layout === 'blank'` và `route.meta.public === true`
